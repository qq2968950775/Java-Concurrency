1：线程并发

串行编程模型的优势在于直观性：每次只做一件事情
如果程序只有一个线程，最多同时在一个处理器上运行，如果有100个处理器，有99个处理器无法使用，

2:线程带来的风险

1:没有充足同步的情况下，多个线程操作执行顺序是不可预测的，因为存在指令重排序的可能
2:线程之间上下文切换会带来额外的开销

当多个线程访问一个可变的状态变量时，没有使用同步，程序就会出现错误
有三种解决方式：
1:线程之间不共享该状态变量
2:该状态变量修改为不可变
3:访问状态变量时需要同步

设计线程安全的类时，需要面向对象的技术、不可修改性、不变性规范

3:什么是线程安全：
正确性，
当多个线程访问某类时，这个类始终都能表现出正确都行为，这个类为线程安全


2.2原子性
竞态条件：不恰当的执行顺序而出现不一样的结果

当某个计算的正确性取决于多个线程交替执行的时候，那么就会发生竞态条件
package com.test.demo.t1;

/**
 * 延迟初始化中的竞态条件
 * 延迟初始化就是说，对象初始化的操作放到实际中运行，只能被初始化一次
 */
public class Test222 {
//竞态条件：如果ab同时执行getInstance,得到不同的结果
    class LazyInitRace{
        private ExpensiveObject instance = null;

        public ExpensiveObject getInstance(){
            if (instance == null){
                instance = new ExpensiveObject();
            }
            return instance;
        }
    }

    class ExpensiveObject{
        
    }
}

解决办法：
保证原子性
1:使用原子变量，保证计数器所有状态都是原子性，尽可能使用线程安全对象AcomicLong

package com.test.demo.t1;

import java.util.concurrent.atomic.AtomicLong;

/**
 * 使用原子性操作
 */
public class Test24 {
    private final AtomicLong count = new AtomicLong(0);
    
    public long getCount(){
        return count.get();
    }
    
    public void service(){
        count.incrementAndGet();
    }
}

2.3加锁机制
光使用Atomic来管理了线程安全的计数器状态，是否能达到线程安全，是否能解决竞态问题？
也不能直接达到
package com.test.demo.t1;

import java.math.BigInteger;
import java.util.concurrent.atomic.AtomicLong;
import java.util.concurrent.atomic.AtomicReference;

/**
 * 使用原子性操作
 */
public class Test25 {
    private final AtomicReference<BigInteger> lastNumber = new AtomicReference<>();
    private final AtomicReference<BigInteger[]> lastFactois = new AtomicReference<>();
    
    public void service(){
        //bigInteger的值从别处获取
        BigInteger i = null;
        if (lastNumber.get().equals(i)){
            System.out.println(lastFactois.get());
        }else{
            BigInteger[] bigIntegers = new BigInteger[i];
            lastNumber.set(i);
            lastFactois.set(bigIntegers);
        }
    }
}

线程安全性的要求：多个线程之间的操作都要保证不变性条件不被破坏
错误的原因：Test25会破坏不变性条件，无法同时更新lastNumber和lastFactois的值，如果只修改了一个变量，其他线程发现不变性的条件被破坏了，不能保证同时获取两个值，线程a获取值，线程b也可以被修改
